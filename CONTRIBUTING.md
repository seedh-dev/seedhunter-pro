# Contributing to SeedHunter Pro

Thanks for your interest in contributing. SeedHunter Pro's core application is
closed-source, but we welcome contributions to documentation, bug reports,
and feature suggestions.

## How to Contribute

### Bug Reports

Found a bug? Open an issue using the
[Bug Report template](.github/ISSUE_TEMPLATE/bug_report.md).

Include:
- Your SeedHunter Pro version (Settings → About)
- Operating system and version
- Steps to reproduce the issue
- Expected vs. actual behavior
- Screenshots if applicable

### Feature Requests

Have an idea? Open an issue using the
[Feature Request template](.github/ISSUE_TEMPLATE/feature_request.md).

Tell us:
- What problem the feature would solve
- How you envision it working
- Whether you've seen this in other tools

### Documentation

Documentation lives in the `docs/` folder. If you find errors, unclear
instructions, or missing information:

1. Fork the repository
2. Make your changes
3. Submit a pull request with a clear description

### Translations

We're exploring multi-language support for documentation. If you'd like to
translate docs into your language, open an issue to coordinate.

## What We Can't Accept

- Source code contributions (the core app is closed-source)
- Pull requests modifying the application binaries
- Requests for features that would compromise user privacy

## Style Guide

- Documentation: Markdown, 80-character line wrapping preferred
- Tone: technical but accessible, no jargon without explanation
- British/American English: either is fine, just be consistent within a file
